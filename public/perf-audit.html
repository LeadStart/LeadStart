1. Card hover shadow stack Highest impact

Every [data-slot="card"] has a 3-layer box-shadow and transitions the entire shadow stack on hover — that's a repaint of all three shadows each frame of the 200ms transition. Dashboards render 10-30+ cards; hovering through them forces repaint cascades.

Fix: paint the hover shadow on a ::after pseudo-element and animate opacity. Browser rasterizes the shadow once, then composites — GPU-only, no repaint.

2. Recharts entrance animation on every re-render High impact

Recharts isAnimationActive defaults to true. Every data change replays the 1.5s entrance animation — and with your SWR prefetch/warm-cache flow, charts can re-animate 2-3× per page load. Not just visually distracting: re-animation forces SVG path interpolation + repaints for the duration.

Fix: add isAnimationActive={false} to each <Area>, <Line>, <Bar>, <Pie>.

◆ Current (replays on every fetch)

<Area type="monotone" dataKey="sent" /> // isAnimationActive defaults true

◆ Optimized (instant paint)

<Area type="monotone" dataKey="sent" isAnimationActive={false} />

3. Button glow — multi-layer shadow + hover transition Medium impact

.btn-blue / .btn-gold stack three shadow layers including a 0 0 22px glow, and the whole stack transitions on hover. On rows with many action buttons (contacts, clients, campaigns tables), hover-sweep triggers repaints.

Fix: single shadow layer + pseudo-element for the hover glow + filter: brightness() for the color shift. Same look, GPU compositing only.

◆ Current (globals.css:259)

box-shadow: inset 0 1px 0 rgba(255,255,255,.25), 0 0 22px rgba(46,55,254,.35), 0 4px 14px rgba(28,36,184,.28); transition: box-shadow 0.2s, transform 0.2s;

◆ Optimized (single shadow + glow pseudo)

box-shadow: 0 4px 14px rgba(28,36,184,.32); transition: transform 0.2s, filter 0.2s; &::after { box-shadow: 0 6px 20px rgba(46,55,254,.42); opacity: 0; } &:hover::after { opacity: 1; } &:hover { filter: brightness(1.1); }

4. Nav active item uses filter: drop-shadow (2×) Low impact — easy win

filter: drop-shadow() is the most expensive CSS filter: it rasterizes the element's alpha mask and blurs it each frame. Only one nav item is active at a time, so total cost is low — but box-shadow on the existing ::before pseudo-element gives identical visuals for zero filter overhead.

5. Avatar ::after uses mix-blend-mode Medium impact

mix-blend-darken on every avatar's border pseudo-element forces a new stacking context + extra compositing pass per avatar. On team/contacts/clients pages with dozens of avatars, this accumulates. Low per-instance; real in aggregate.

Fix: use a solid semi-transparent border color. Visually indistinguishable at typical avatar sizes.

◆ Current (avatar.tsx:20)

&::after { border: 1px solid var(--border); mix-blend-mode: darken; /* new stacking context */ }

◆ Optimized (semi-transparent border)

border: 1px solid rgba(15, 23, 42, 0.12); /* no new context */

6. Every chart mounts its own ResizeObserver Low impact — architectural

Recharts' <ResponsiveContainer> creates one ResizeObserver per chart. Pages with 4+ charts (client/reports, client/activity) stack observers firing on every layout shift. Not catastrophic, but wasteful.

Fix: share a single observer via a useResizeObserver hook at the parent, or use fixed-width charts when possible. Lowest priority unless a page mounts many charts.

7. Gradients — not a concern No impact

Linear and radial gradients are GPU-rasterized once per element and cached. Sidebar gradient, button gradient, card gradient — all essentially free. Don't optimize these.

Recommended order

1️ Chart animation HIGH

One-line fix per chart: isAnimationActive={false}. Eliminates SWR re-animation cascade. Biggest perceptible win.

2️ Card hover shadow HIGH

Migrate to pseudo-element opacity fade in globals.css:229. Affects every card on every page.

3️ Button glow stack MED

Same pseudo-element pattern for .btn-blue / .btn-gold. Helps on tables with many action buttons.

4️ Avatar mix-blend MED

Swap mix-blend-darken for a rgba border. One-line change in avatar.tsx:20.

5️ Nav drop-shadow LOW

Move the two drop-shadows to box-shadow on the existing ::before. Free visual equivalence.

6️ Chart observers LOW

Only worth doing if a single page renders many charts. Architectural, defer unless measured.
